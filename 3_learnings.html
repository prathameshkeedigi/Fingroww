Next.js 14 App Router Architecture
Deeply understood the App Router model: Server Components for data fetching (no useEffect), Client Components only for interactivity. Built 11 server-rendered pages with a centralised getFinancialSnapshot() function doing 6 parallel Supabase queries — all computed before the first byte reaches the browser.

🗃️
Supabase & PostgreSQL with RLS
Designed 6 database tables with full Row Level Security — USING and WITH CHECK clauses on every table. Learned how database-layer isolation differs from application-layer auth guards. Used GENERATED ALWAYS AS computed columns, UNIQUE constraints, and compound indexes for performance. Auth via getUser() (server-validated JWT), never getSession().
SupabasePostgreSQLRLSMigrations
🔐
Secure Authentication & Middleware
Implemented route-level protection using Next.js middleware with Supabase Auth JWT validation. Learned the security difference between getUser() (server-validated token) vs getSession() (client-provided cookie). Built redirect logic: new users → onboarding; returning users → dashboard. All 11 routes protected.
Supabase AuthJWTMiddlewareRoute Guards

Financial Calculator Design
Built 5 independent financial calculators from scratch: Debt Avalanche (sorts by interest rate, computes total months to payoff + interest), Goal Planning (SIP formula + probability score from surplus), Emergency Fund (employment-type-aware targets), Health Score (4-component weighted 0–100), and Insights Engine (11 rules, priority sorting).

🤖
AI Integration with Groq + Llama 3.3-70B
Integrated Groq's inference API with Llama 3.3-70B to generate monthly AI financial analysis. Key learning: LLMs hallucinate numbers — so the AI receives only pre-computed values with strict instructions ("Do NOT calculate"). Built a monthly caching system in PostgreSQL so the API is called only when data changes, not on every page load.
Groq SDKLlama 3.3-70BPrompt EngineeringAI Caching

Data Visualisation with Recharts
Built multiple chart types: a delta bar chart with direction-aware colour coding (indigo = improved, rose = worsened based on financial meaning, not just sign), a net worth sparkline, a health score gauge, and surplus progress bars. Learned how to use ReferenceLine, Cell-level colour logic, and ResponsiveContainer for adaptive layouts.

Next.js Server Actions
Replaced traditional REST API endpoints entirely with Server Actions. Built 9 server action files for auth, profile, debts, goals, assets, monthly check-in, history, snapshot, and AI summary. Learned how mutations work without an API layer, how to revalidate server caches after mutations, and how to handle errors from server actions in client components.

Test-Driven Development with Jest
Wrote 647 tests across 28 test suites covering all 5 financial calculators and the full getFinancialSnapshot data flow integration. Tested edge cases: zero income, negative surplus, missing check-in, stale savings, debt-free users. Also wrote Playwright E2E tests for auth and onboarding flows. Learned the difference between unit, integration, and end-to-end test layers.
JestPlaywrightUnit TestsE2E Tests

IST Timezone & Indian Financial Year
Built timezone-aware date utilities (istYearMonth, istMonthKey) to handle a subtle bug: a UTC midnight on the 1st can show as the last day of the previous month in IST (+5:30). Implemented the Indian Financial Year (April–March) for the History module. Learned how incorrect timezone handling corrupts financial records in subtle, hard-to-debug ways.

Learned how to break a complex system (11 modules + AI + 647 tests) into deliverable increments without breaking existing features.
