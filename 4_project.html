Fingroww — Personal Finance Dashboard

A full-stack personal finance operating system for Indian working professionals — combining a Financial Health Score, debt payoff simulation, goal planning, emergency fund tracking, net worth analysis, AI recommendations, and 6-month delta charts in one authenticated dashboard.

📋 Problem Statement

Young Indian professionals earning ₹30,000–₹1,50,000/month face a consistent problem: they have income but no structured system to understand where they stand financially. Most existing apps are either expense trackers (Walnut, Money Manager) or investment platforms (Groww, Zerodha) — none provide a unified financial health assessment across savings, debt, goals, and emergency preparedness simultaneously.

Key gap: No single free platform gives Indian professionals a real-time, data-driven financial health score with specific, personalised action steps — all in one authenticated dashboard.

Users carry high-interest credit card debt while their savings sit idle, missing the opportunity to pay it off faster. They set financial goals without knowing whether they're realistic given their actual surplus. They have no emergency fund coverage estimate calibrated to their employment type and number of dependents. Financial decisions are made with incomplete information.

🎯 Project Objectives

  • Design and develop a unified, authenticated personal finance dashboard for Indian professionals aged 22–40.
  • Implement a Financial Health Score (0–100) computed across four components: savings rate, debt-to-income ratio, emergency fund coverage, and income level.
  • Build a monthly check-in system so users can enter real actuals (income, expenses, bank balance) to override profile estimates — making all calculators use real data.
  • Simulate debt payoff timelines using the mathematically optimal avalanche strategy, with an interactive extra payment slider.
  • Provide goal planning with SIP-based required monthly investment and probability scoring based on the user's actual available surplus.
  • Implement an AI financial analysis module using Groq + Llama 3.3-70B that generates 3 specific, personalised recommendations per month — with pre-computed numbers only, never AI arithmetic.
  • Build historical tracking across the Indian Financial Year (April–March) with a delta bar chart comparing any two filled months.
  • Enforce Row Level Security at the database layer — not just application layer — for complete user data isolation.

📌 What Makes Fingroww Different

1
One snapshot, all modules. A single getFinancialSnapshot() function runs 6 parallel DB queries + 5 calculators and returns one object. Every page draws from this — no duplicated fetches, no inconsistent numbers across tabs.
2
Active vs. Estimated values. If the user has done a monthly check-in, real actuals are used everywhere. If not, profile estimates are the fallback. The distinction is transparent to the user via a "Last updated" badge.
3
AI that never does math. Llama 3.3-70B receives only pre-computed numbers. The prompt explicitly says "Do NOT calculate." This eliminates LLM hallucination on financial figures — the most common failure mode of AI finance tools.
4
Database-layer security. All 6 tables have Row Level Security with USING + WITH CHECK. Even a compromised application layer cannot read another user's data — the database rejects the query.
5
Indian context built-in. IST timezone handling, Indian Financial Year (April–March), employment-type-aware emergency targets, INR formatting with lakh/crore notation, 80C tax deduction gap detection for salaried users.

🏗️ System Architecture — 4-Layer Design

Fingroww is a server-first Next.js 14 App Router application. There is no separate backend server — data fetching and mutations happen via Server Components and Server Actions directly connected to Supabase PostgreSQL.

Presentation Layer
Next.js 14 App Router Server Components (data) Client Components (interactivity only) Tailwind CSS (dark theme) Recharts (charts) Lucide Icons
↓ Server Actions (9 action files — no REST endpoints)
Application Layer
getFinancialSnapshot() — 6 parallel queries Debt Avalanche Calculator Goal SIP Calculator Emergency Fund Calculator Health Score (0–100) Insights Engine (11 rules) Groq AI Summary
↓ Supabase JS Client (server-side, server-validated JWT)
Data Layer
Supabase PostgreSQL financial_profiles debts goals assets monthly_records monthly_snapshots RLS on all 6 tables
↓ Groq API (Llama 3.3-70B) for AI Financial Analysis
Integration Layer
Supabase Auth (JWT) Groq SDK Next.js Middleware (getUser validation) Vercel Deployment

🔒 Auth & Security Model

Every request passes through Next.js middleware which calls supabase.auth.getUser() — this validates the JWT on the Supabase server, not from the browser cookie. This prevents session cookie spoofing. Unauthenticated users are redirected to /login. Users without a profile are redirected to /onboarding.

Security is enforced at two independent layers: middleware (application) + Row Level Security (database). A broken app cannot leak another user's data.

All 6 tables use RLS policies with both USING (controls SELECT/DELETE) and WITH CHECK (controls INSERT/UPDATE). The policy is always auth.uid() = user_id. No service-role key is exposed to the frontend.

📊 Data Flow — Dashboard Load

1
User visits /dashboard → Next.js middleware validates JWT via getUser() → passes to DashboardPage (Server Component)
2
getFinancialSnapshot(userId) → 6 parallel Supabase queries: profile, debts, goals, assets, monthly_record (this month), snapshots (last 6)
3
Active values resolved: active_income / active_expenses = checkin values OR profile estimates. active_savings = checkin.savings_balance OR profile.savings_balance (most recent, no month gate)
4
5 calculators run in sequence: debt payoff → goal SIP → emergency fund → health score (needs all prior) → insights engine (needs all prior)
5
Full snapshot returned → Dashboard UI renders with health gauge, net worth, surplus, insights, goals, AI analysis
6
SnapshotSaver client component fires saveSnapshot() via useEffect → upserts monthly_snapshots (including ai_summary: null to clear stale cache) → generateAISummary called → Groq returns 3 recommendations → stored in DB

🧩 Core Functional Modules

Module 01
Authentication & Onboarding
Email/password login via Supabase Auth. Middleware protects all routes. One-time onboarding form captures income, expenses, savings, employment type, risk profile, age, dependents.
Module 02
Dashboard
Health score gauge, net worth sparkline, monthly flow card (income/expenses/EMIs/surplus), top 3 insights, goals progress bars, AI analysis tiles. Auto-saves snapshot on every load.
Module 03
Debt Management
Add/edit/delete debts (6 categories). Avalanche payoff simulation: ordered by interest rate, shows months to debt-free + total interest. Extra payment slider persisted to profile.
Module 04
Goal Planning
Create goals with target amount, date, expected return, priority. System computes required SIP and probability of success based on available surplus after higher-priority goals.
Module 05
Emergency Fund
Coverage months vs. target (employment-type + dependents aware). Status: Funded / Partial / Danger. Shortfall amount + build plan with milestones given current monthly surplus.
Module 06
Net Worth
Assets CRUD across 9 categories (bank savings, FD, mutual funds, stocks, PPF, EPF, gold, real estate, other). Net worth = total assets − total active debt. Delta from last snapshot.
Module 07
Profile & Monthly Check-in
Edit profile estimates. Enter actual income, expenses, bank balance for this month. Once submitted, all calculators switch from estimates to real monthly data instantly.
Module 08
History & Delta Chart
Indian FY month grid (April–March). Enter actuals for any past month. Delta compare: pick any 2 filled months → 4 metric cards + direction-aware colour-coded bar chart (Recharts).
Module 09
Insights Engine
11-rule rule engine evaluates every session. Up to 8 prioritised insights (Critical → High → Medium → Low) with action links to the relevant module. Health score component breakdown bars shown.
Module 10
AI Financial Analysis
Groq + Llama 3.3-70B generates 3 specific monthly recommendations. AI receives pre-computed numbers only — never calculates. Cached in monthly_snapshots, regenerated when data changes.
Module 11
Invest & Borrow
Invest: SIP calculator with 80C deduction gap flag for salaried users. Borrow: Loan simulator — enter any loan parameters, see EMI, post-loan surplus, and new DTI (colour-coded: safe/caution/danger).

📄 Conclusion

Fingroww successfully implements a comprehensive personal finance operating system for Indian professionals — combining secure authentication, real-data monthly check-ins, 5 independent financial calculators, a rule-based insights engine, AI-powered analysis, and historical trend visualisation in one unified, server-rendered dashboard.

Built with Next.js 14, Supabase (PostgreSQL + RLS), Groq + Llama 3.3-70B, and validated by 647 automated tests — Fingroww demonstrates how modern web technologies can deliver enterprise-grade financial software for everyday users.

The project directly aligns with SDG 8 (Decent Work and Economic Growth) by giving Indian professionals the tools to understand their financial position and make data-driven decisions — reducing debt stress, building emergency savings, and planning achievable financial goals.
